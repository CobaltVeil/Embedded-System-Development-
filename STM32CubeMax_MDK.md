### 1 在STM32CubeMax配置EXTI外部中断项目中，出现设置抢占优先级，造成反向中断嵌套调用造成程序卡死的问题。
系统默认生成System Tick Timer的抢占优先级为15，EXTI 0 Line一般由用户自行设置，在外部中断回调函数中如果写：
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    if(GPIO_Pin == USER_KEY_Pin)
    {
        HAL_Delay(10);
        HAL_GPIO_TogglePin(GREEN_LED_GPIO_Port, GREEN_LED_Pin);
    }
}
HAL库延时函数HAL_Delay使用的是系统滴答定时器作为时间基准，而系统滴答定时器同为中断，在外部中断中触发系统滴答定时器中断会涉及中断优先级的问题，
结论：尽量不要在外部中断回调函数中调用低优先级中断函数，在外部中断回调函数中尽量少的去执行功能，可以通过设标志位再在主循环中执行的方式来处理。

### 2 在STM32CubeMax配置定时器项目中，如何根据定时时间计算预分频寄存器、自动重装载寄存器的值。
#### 2.1 关于定时器所在总线频率与预分频寄存器、自动重装载寄存器、预计定时时间几者之间的关系为 ：
T = (PSC+1)(ARR+1)/APBX ，上述公式意味着根据定时器所在APB总线频率进行规划，考虑到ARR(自动重装载寄存器)的最大计数取决于所用定时器的位数(目前使用平台有16位和32位)，2的16次方为65536，即ARR选值不能超过(65536-1)。以此为前提APBX/(PSC+1)得到的是定时器计数频率，代表多长时间记一次数(APBX为84MHZ时，(PSC+1)为84时,为1000000HZ，代表1us计数一次；(PSC+1)为84000时,为1000HZ代表1ms计数一次，)。根据需要定时的时间来参考(ARR+1)的值，如想要定时500ms时((PSC+1)为8400，APBX为84MHZ)，得到(ARR+1)的值为5000，从而实现500ms定时。

#### 2.2 这里我提出一个问题，定时时间一定，APB总线频率一定的情况下，预分频寄存器和自动重装载寄存器各自的值越大或者越小有什么效果？
PSC（预分频器）和 ARR（自动重装载值）的分配，本质上是在时间精度和系统资源之间做权衡，简单来说是这样的：
PSC越小，ARR越大：定时精度越高，但需要频繁处理中断或占用更大的计数器范围。
PSC越大，ARR越小：定时精度越低，但更节省CPU资源（中断次数少）或适应小容量定时器。
##### 补充，PSC与ARR值关于中断占用的误解
在以上PSC与ARR权衡中，关于定时精度对应中断频繁程度的情况，定时精度高意味着计数频率高，定时精度低意味着计数频率低没有问题，而是否频繁占用处理中断取决于是什么类型的中断。溢出型中断因为定时的值是一定的，只在定时到达时溢出产生中断，所以中断次数固定不受定时精度影响。非溢出功能（比较/捕获/PWM）的触发频率与计数频率正相关，定时精度高时，每个比较点都需要处理一次中断，CPU比较忙；而定时精度低时，比较点分的比较开，可选比较点少，更节省CPU资源。
